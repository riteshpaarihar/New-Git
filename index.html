Git Command

Git Installtion and config

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

After installation, open a terminal or command prompt and configure Git with your name and email address. This information will be used to identify your commits.

Git Create new branch

git branch
git checkout "ritesh"
git checkout -b "emBug"
git checkout master

git branch: This command lists all existing branches in the repository. It does not create a new branch. It's useful for checking which branches already exist before creating a new one.
git checkout "ritesh": This command switches to the existing branch named "ritesh". It doesn't create a new branch but rather checks out an existing one.
git checkout -b "emBug": This command creates a new branch named "emBug" and switches to it. The -b option is used to create a new branch, and "emBug" is the name of the new branch.
git checkout master: This command switches back to the "master" branch. It's useful for moving between branches during development.

git commit

git init
git add index.html
git add .
git add *
git commit -m "my first commit"

git init: This command initializes a new Git repository in the current directory. It creates a hidden .git directory that contains the repository data and metadata.
git add index.html: This command stages the file index.html for the next commit. Staging means that Git is tracking changes in this file, and it will be included in the next commit.
git add .: This command stages all changes in the current directory and its subdirectories for the next commit. The dot (.) represents the current directory.
git add *: This command stages all changes in the current directory (excluding hidden files) for the next commit. The asterisk (*) is a wildcard that matches all files and directories.
git commit -m "my first commit": This command creates a commit with the changes staged in the previous steps. The -m option allows you to provide a commit message inline. In this case, "my first commit" is the commit message describing the changes made in this commit.
